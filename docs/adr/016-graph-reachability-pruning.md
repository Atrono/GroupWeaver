# ADR-016: Graph-Layer Reachability Pruning (node-Refresh orphan residual)

**Status:** Proposed (adoption deferred — see Decision D7) · **Date:** 2026-06-21
**Refines:** ADR-004 D1 (totality) / ADR-005 D4 (the accepted residual) / ADR-009 D6 (pure projector) · **Decides:** issue #54 · **Phase:** 4 (a phase-3 follow-up)

## Context

`DirectorySnapshot` is append-only by contract (no `RemoveObject`; data-model rule): `SetMembers` REPLACES a parent's members (refresh semantics, stale edges vanish) but the ex-member OBJECTS stay in `Objects`. `GraphBuilder.Build(snapshot, rootDn)` is a pure, total topology projector (ADR-004 D1): every object becomes a node and every membership/containment edge endpoint is materialized — *edges are never dropped*. Aggregation lives engine-side so the builder never consults the report (ADR-009 D6).

Consequently, after a node-level **Refresh** (ADR-005 D4 — forced re-expand of one selected node) drops a group's member, the ex-member's node lingers. ADR-005 D4 split the residual precisely: an **in-scope** ex-member keeps a still-true **containment** edge (so it stays connected), while an **expanded out-of-scope** ex-member loses its only (membership) edge and lingers as a disconnected node on the outer External ring. ADR-005 D4 knowingly accepted this for v0.1.

Whole-scope **Reload** (#30, ADR-005 addendum) already cures it *by construction*: `ReloadScopeCommand` re-runs `LoadScopeAsync(RootDn)` → fresh snapshot → rebuild → re-evaluate, so there are no ex-member objects and no divergence — but it resets the viewport (honest: topology is wholesale-new). #54 asks whether a lighter, in-place **graph-layer pruning** (drop the disconnected node without a full reload) is worth it. ADR-005's addendum flagged that such pruning needs its own ADR because it would break `GraphBuilder` totality (ADR-004 D1 / ADR-009 D6) and create graph-vs-report node-set divergence (a pruned node could still carry a finding the AP 3.4 sidebar lists). This ADR is that design.

## Decision

1. **Snapshot stays append-only; pruning is a GraphBuilder-layer projection filter, opt-in.** No `RemoveObject` is ever added (non-negotiable, data-model: removing a loaded object tears the null-vs-empty / `IsLoaded` tri-state and resurrects `MembershipTraversal` frontier nodes). Pruning lives entirely in `GraphBuilder`, reading the snapshot and never mutating it, behind a new `GraphLayoutOptions.PruneUnreachable` flag **defaulting to `false`**. With the flag off, `Build` output is byte-identical to today — ADR-004 D1 and ADR-009 D6 hold unchanged for every existing call site and test.

2. **Prune = not reachable from the root over the materialized edge set (both kinds).** After `Build` materializes the full node + edge set (totality intact at this stage), compute **undirected reachability from `rootDn`** over `GraphModel.Edges` — *both* `member` and `contains` edges — and keep only the root plus nodes connected to it; drop the rest, along with any edge incident to a dropped node (never a dangling edge). `rootDn` is already a `Build` parameter (passed from `WorkspaceViewModel.RootDn` on every call) — no new plumbing, no new wire fields; the pruned `GraphModel` serializes through the existing `GraphJson` choke point unchanged (just fewer nodes/edges). Undirected connectivity over *both* edge kinds is required so an in-scope ex-member that retains a valid `contains` edge is **kept** (ADR-005 D4), and only the truly disconnected out-of-scope residual is pruned.

3. **This is graph connectivity, NOT a membership walk.** Reachability here is over the built `GraphEdge` set (member ∪ contains), seeded at the root — a layout/projection concern. It is deliberately distinct from `MembershipTraversal.Walk` (membership-only, group→member, the sole sanctioned *transitive membership* walk per ADR-006 / data-model) and does not touch that mandate. The pass is O(V+E) over the already-materialized local edge set: `snapshot.Edges` is still read **exactly once** (perf contract), with no second read and no per-node `GetMembers` loop.

4. **The tri-state is preserved — "unexpanded areas are unchecked" still holds.** Pruning operates only on the materialized edge graph, which contains only *loaded* relationships. An unexpanded parent (`GetMembers == null`, no outgoing edges) is never expanded or emptied by pruning; a freshly-loaded root with no members is the reachability seed and is kept. During normal load/expand, every node reached the graph *by* being expanded from the root, so it is reachable and pruning is a **no-op**. Pruning bites only when a Refresh has dropped a node's last edge — exactly the ADR-005 D4 residual.

5. **Totality becomes a documented, flag-gated exception.** With `PruneUnreachable` on, the `GraphBuilder` contract reads "total over the root-reachable sub-graph" instead of "total over `snapshot.Objects`". This is the single, explicit exception to ADR-004 D1 / the class-summary "edges never dropped" contract, recorded here and to be noted in the `GraphBuilder` summary. The default (off) remains the proven pure projector.

6. **Graph-vs-report divergence is made VISIBLE and bounded, never silently hidden.** `RuleEngine`/`RuleReport` are **unchanged** (ADR-009 D6 preserved — the engine stays pure over the full snapshot; aggregation engine-side). The findings sidebar continues to list **all** of `RuleReport.Violations` — no finding is ever dropped (honesty is the product's ethos). A finding whose anchor DN was pruned stays in the sidebar; its jump-to-node is *already* a typed no-op (`JumpAsync`→`FocusAsync([dn])` silently skips an off-graph/unknown DN — existing behavior, never an error). To keep that honest rather than mysterious, pruning that removes ≥1 node MUST surface a transparent indicator — e.g. *"N node(s) hidden as disconnected — Reload to rebuild"* — and pruned-anchor sidebar rows are flagged non-jumpable. We do NOT prune the report in lockstep: doing so would either silently drop real findings or force the engine to consume graph reachability (violating ADR-009 D6). The divergence is therefore bounded (only disconnected, stale nodes) and disclosed, not eliminated.

7. **Adoption is DEFERRED; this ADR is Proposed, not Accepted.** Reload (#30) is a one-click, divergence-free, fully-correct cure; the residual is cosmetic (one disconnected stale node until the next Reload). The cost of pruning — the first exception to totality, a visible graph-vs-report divergence, and two new no-overlap proofs to maintain (D8) — is **not justified today**. Adopt only if user feedback shows the node-Refresh-without-Reload residual is a recurring pain. Until then ADR-005 D4 + Reload stand; this document is the ready-to-build design, gated on that feedback.

8. **Geometry no-overlap must be re-proved for the pruned model (on adoption).** Pruning runs *before* `PlaceNodes`, so positions are computed on the pruned node set; the concentric ring key (relative DN depth below root, kind rank) is unaffected for survivors, and removing nodes cannot introduce overlaps. The proof is nonetheless re-established: add a `GraphBuilderGeometryTests` case that builds a post-Refresh snapshot with `PruneUnreachable` on and asserts `MinPairwiseCenterDistance ≥ D` *and* that the ex-member DN is absent; extend `tests/graph-bundle/verify.mjs`'s render-space no-overlap assert (`MIN_CENTER_DISTANCE = 44`) to a pruned fixture. Pruning is deterministic (`Dn.Comparer`-keyed reachability over a stable edge set), so both proofs are stable.

## Consequences

- **Pro:** in-place declutter of the post-Refresh orphan without a full Reload or viewport reset; opt-in, so zero impact on the default build, existing contracts, and existing tests.
- **Con:** introduces the first documented exception to `GraphBuilder` totality (ADR-004 D1); introduces a bounded, *disclosed* graph-vs-report divergence (D6); adds a reachability pass plus new C# + Playwright no-overlap proofs to maintain.
- **Net:** Reload already solves the residual completely and without divergence. Pruning is a strictly narrower, more complex alternative whose only advantage is avoiding the viewport reset — hence Proposed-and-deferred (D7).

## Rejected alternatives

- **`DirectorySnapshot.RemoveObject` (mutate the snapshot).** Rejected: breaks append-only; tears the null-vs-empty / `IsLoaded` tri-state (a removed loaded parent reads as never-loaded → the "unexpanded = unchecked" lie) and resurrects `MembershipTraversal.Walk` frontier nodes (data-model rule).
- **Orphan-only pruning (drop degree-0 nodes).** Rejected as incomplete: misses disconnected *multi-node* islands (an ex-member that had its own expanded children is not degree-0). Reachability-from-root (D2) subsumes it.
- **Prune the `RuleReport` in lockstep (eliminate divergence).** Rejected: either silently drops real findings (dishonest) or forces the engine to consume graph reachability (violates ADR-009 D6 pure-over-snapshot). Divergence is instead made visible and bounded (D6).
- **Directed (membership-only) reachability.** Rejected: would prune in-scope ex-members that retain a valid `contains` edge (ADR-005 D4). Undirected connectivity over *both* edge kinds (D2) is correct.
- **Always-on pruning (no flag).** Rejected for v0.x: a global change to the D1 totality contract. Opt-in keeps the default the proven pure projector and confines the divergence to a deliberate mode.
- **Status quo only (close #54 as wont-fix).** Not chosen: the design is cheap to record and may be wanted later; deferring adoption (D7) keeps the option open without paying its cost now.
