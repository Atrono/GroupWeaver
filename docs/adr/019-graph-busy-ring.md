# ADR-019: Graph-layer in-canvas busy ring — the lazy-expand round-trip cue

**Status:** Accepted · **Date:** 2026-06-22
**Refines:** ADR-017 (which shipped F1 + F2 and **split F12 out** — its Consequences note: F12 "needs `IGraphRenderer.SetBusyAsync` as a default no-op method … dispatched **without** `EnterSingleFlight`/`_commandInFlight` … recording on its own channel") · ADR-004 D5 (renderer seam) · ADR-005 D3 (lazy-expand pipeline) · ADR-010 D1 (severity overlay channel) · **Decides:** issue #94 (the F12 in-canvas busy ring) · **Phase:** 4 (v0.2 polish pass)

## Context

ADR-017 reintroduced the animation ADR-005 deferred, but scoped to F1 (new-node enter fade) + F2 (eased focus-fit), and **split F12** — the in-canvas busy cue — to this follow-up for two structural reasons it called out: F12 needs an `IGraphRenderer` interface change (so every renderer implementation and the test fake compile), and a fire-and-forget busy signal must **not** contend the renderer's shared single-flight, because the very pipeline that raises busy=on (the AP 2.3 lazy expand, ADR-005 D3) is already holding that guard for its `UpdateGraphAsync`/`FocusAsync`. Re-entering `EnterSingleFlight` from inside that window throws — the busy signal would crash the expand it is supposed to decorate.

The gap F12 closes: a lazy-expand fetch is a directory round-trip with no in-canvas feedback. The native chrome shows `IsLoading`, but the eyes are on the graph node being double-tapped. ADR-017 added motion to the *result* of an expand; F12 adds a cue to its *latency*.

## Decision

1. **`SetBusyAsync(string dn, bool on, …)` as a default no-op interface method** on `IGraphRenderer`, mirroring `ShowDiffGraphAsync`/`ExportPngAsync` — so `FakeGraphRenderer` and every implementation compile without an override, and only the real `CytoscapeGraphRenderer` overrides it. Toggles the in-canvas busy ring on the node named `dn`.

2. **Dispatched fire-and-forget, off every shared channel.** The real override does **not** call `EnterSingleFlight`/touch `_commandInFlight` (the in-flight expand holds the guard; re-entry would throw and deadlock the pipeline), does **not** arm a confirmation TCS (never the 60 s `BridgeTimeout`, never the `focused` channel), and does **not** block on readiness. Instead of `TryAwaitReadyAsync` (which would 60 s-block and raise on timeout), it does a **non-blocking** ready check — `_webView is null || !_ready.Task.IsCompletedSuccessfully` → return — so a busy racing ahead of the bundle simply no-ops; the next `graphUpdate` clears the transient flag regardless. It is **never-throw** (the async-void `RelayCommand` expand path has no handler): a degraded bridge surfaces as `RendererError` and returns normally. The `busy` command is serialized through `GraphJson` (a `BusyDto` record beside `FocusDto`) so a comma/quote-containing DN is escaped exactly as a focus DN is.

3. **graph.js `node[busy][!sev]` overlay**, `#4FA3E3` / opacity `0.35` / padding `8`, **appended after** the severity (`node[sev=…]`) and roll-up (`node[below]`) rules. The `[!sev]` gate means a flagged node's own severity halo always wins the shared `overlay-*` channel — busy never paints over a finding. **Static** — no per-frame tween (software-rendering-floor safe per `lab-environment.md`; a pulsing animation is deferred).

4. **Transient `busy` data flag, cleared automatically.** The `busy` command sets `node.data('busy', true)`; off uses `removeData('busy')` (NOT `data('busy', false)`) so a cleared node is byte-identical to a never-busy node under the `node[busy]` selector. The next `graphUpdate`'s `cy.batch` remove-all/add-all drops the flag for free — there is **no** explicit clear command and no `busy` field in `takePendingElements` (the flag is transient and live-only). A `busy` arriving before any `graphCommit` (`cy === null`) silently breaks — unlike `graphUpdate`/`exportPng`, it raises **no** `jsError`, keeping the `verify.mjs` zero-jsError audit green for a benign pre-commit race. Lookups use `getElementById` only (comma-DN safe, ADR-004 D5).

5. **Wired in `ExpandAsync` on the FETCH PATH ONLY.** The cache-hit/focus-only branch returns before any provider call, so it has no round-trip to mark. On the fetch path: `busySet = true; await renderer.SetBusyAsync(dn, true, ct)` is issued **before** the provider round-trip (after `LoadError = null`, before `GetObjectAsync`/`GetMembersAsync`); the `finally` clears it `if (busySet)` with `CancellationToken.None`, so a cancelled or failed expand still drops the ring (the on-call used the real `ct`; the off-call must not be cancelled out from under the clear).

## Consequences

- The lazy-expand directory round-trip now shows an in-canvas cue on the node being expanded, complementing ADR-017's motion on the expand *result*. The native-chrome `IsLoading` indicator is unchanged.
- **Channel-collision, accepted:** the busy overlay and the `node[below]` roll-up ring both ride the cytoscape `overlay-*` channel. `[!sev]` excludes only the node's **own** severity, not `below`. So a node that is below-only (descendants flagged, itself clean) could show busy with `overlay-padding 8` < `node[below]`'s `10` — the wider below ring still reads through, so the roll-up cue is not lost; accepted (the overlap is momentary and only during an in-flight expand of an already-roll-up node).
- `verify.mjs` (owned by the test-engineer) gains busy-overlay-set, clear-on-`graphUpdate`, and motion-counters-untouched (`__gwAnimateCalls`/`__gwEnterAnims` from ADR-017 must not move on a busy toggle) asserts. The C#↔JS `#4FA3E3` is already pinned by the `sev=info` overlay, so `Graph_PaletteMatchesAdObjectKindConverters` is unaffected by reusing it here.
- `FakeGraphRenderer` needs a `SetBusyAsync` override recording its own channel (never `FocusCalls`) for the `WorkspaceViewModel` expand tests to assert the busy=on-before-fetch / busy=off-in-finally ordering — a `tests/` change owned by the test-engineer.
