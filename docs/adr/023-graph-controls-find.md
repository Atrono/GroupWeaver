# ADR-023: In-graph control cluster + find-a-node (web layer)

**Status:** Accepted · **Date:** 2026-06-22
**Refines:** ADR-004 (the graph bundle / `getElementById`-only comma-DN safety, D5) · ADR-017 (eased camera motion; reduced-motion degrade) · ADR-018 (the `min-zoomed-font-size` label gate; `applySelection`/neighborhood dim) · ADR-020 (selection sync; the `focused` confirmation is the .NET focus protocol's, not anyone else's) · **Relates:** ADR-001 (airspace — graph controls are web-layer, never native over the HWND) · **Decides:** the WP-B in-graph controls + find work package · **Phase:** 4 (post-v0.2 UX)

## Context

A dense node-link view has no way to **find** a node or reset the camera. With ~200 demo objects the only "find" is jump-to-a-FLAGGED-node via the violations sidebar; there is no answer to *"where is `DL_App-CRM_RW`?"*, no Fit/reset-view, no zoom buttons, and at fit-zoom the overview is a sea of unlabeled dots (the ADR-018 label gate hides every non-root/non-Error label). ADR-022 deliberately left these out as a **web-layer** package — native controls can't float over the WebView2 HWND (ADR-001), and the legend already proves the pattern: a `position:fixed` overlay *inside* the bundle. This ADR adds an interactive control cluster and find, entirely in `src/App/web/*` — **no C# change** (find reuses the existing `nodeClick` message the renderer already handles).

## Decision

1. **A `#controls` web-layer overlay**, sibling of `#legend` in `index.html`, bottom-right, **`pointer-events:auto`** (the first interactive bundle overlay — the legend is `pointer-events:none`), styled from the same hand-mirrored BrandTokens the legend uses. It holds: **Fit**, **Zoom +**, **Zoom −**, a **Find** text input, and a **Labels** toggle. It lives inside the WebView, never native over it — ADR-001 airspace is about *native* airspace, so an in-bundle interactive overlay is sanctioned (same surface as cytoscape itself).

2. **Zoom / Fit are pure cytoscape, no bridge.** Zoom ± call `cy.zoom({ level: cy.zoom()*f, renderedPosition: viewport-center })` (anchored at center) clamped to cytoscape's min/max; Fit calls `cy.fit(cy.elements(), 80)` (the same 80px padding as `focusOn`). They emit **no** bridge message and touch **no** confirmation channel — they are local view operations.

3. **Find mirrors a tap, then frames LOCALLY — and never emits `focused`.** On submit, a case-insensitive lookup over `cy.nodes()` by **label (Name) OR id (DN)** (exact-or-substring; first match wins, deterministic by the cytoscape z-order) yields a node. Find then: (a) `window.bridge.send({type:'nodeClick', id, label, kind})` so .NET sets `SelectedDn` and the detail panel updates — the exact message a tap sends; (b) `applySelection(node)` for instant selection + neighborhood dim (ADR-018); (c) a **local** eased camera frame (`cy.animate({fit:{eles:node, padding:80}}, {duration:280, easing:'ease-out-cubic'})`, reduced-motion ⇒ instant `cy.fit`). Step (c) deliberately **does NOT** reuse `focusOn()` and **never sends a `focused` bridge message**: that confirmation belongs solely to the .NET-initiated `focus` command's TCS (ADR-017/ADR-020) — an unsolicited `focused` could resolve a stale focus await and would perturb the `JumpAsync`/`FocusCalls` pin. No-match ⇒ a quiet inline "no match" affordance in the cluster; no bridge traffic.

4. **Labels toggle overrides the ADR-018 fit-zoom gate.** Two states — **auto** (default: the base `min-zoomed-font-size:10` gate, root + Error forced on per ADR-018 D4) and **all** (every label visible at fit zoom). "all" is applied by toggling a `gw-labels-all` class on `cy.nodes()` whose rule sets `min-zoomed-font-size:0`; the mode is module-level state **re-applied in `sendLoaded`** so a `graphUpdate` (lazy expand) keeps the chosen mode. No "off" state — it would fight the ADR-018 always-on-root/Error mandate for no real gain.

5. **Web-layer keyboard, scoped to the bundle.** A `document` keydown listener in `graph.js` handles **Ctrl+F** (focus the Find input, `preventDefault`), **Ctrl+0** (Fit), and **+ / −** (Zoom) — only while the WebView has focus, so they never collide with the native ADR-022 shortcuts (F11/Esc/Ctrl+B) which fire when Avalonia chrome has focus. **Esc** in the Find input clears it and blurs.

## Consequences

- New day-to-day navigation: find any node by name/DN, reset/zoom the camera, and read the overview with all labels on — without leaving the graph. The hairball (ADR-004's accepted bezier-edge density) is now *navigable* (find + focus-dim) without re-laying-out the pinned concentric layout.
- **No C# / renderer change.** `IGraphRenderer`, `CytoscapeGraphRenderer`, and the wire contract are untouched; find rides the existing `nodeClick` path, so `SelectedDn`/detail-panel behavior is identical to a tap. The `focused` protocol and the ADR-020 `FocusCalls`/`SelectCalls` pins are untouched (decision D3).
- **Tests (test-engineer):** `tests/graph-bundle/verify.mjs` gains asserts — the `#controls` cluster renders; Fit/Zoom move the camera (`cy.zoom`/`extent` change) without touching the `__gwAnimateCalls`/`focused` counters; Find selects + frames a node by Name and by DN (incl. a comma-DN via `getElementById`-safe lookup) and emits exactly one `nodeClick` and **zero** `focused`; the Labels-all toggle drops `min-zoomed-font-size` to 0 and survives a `graphUpdate`; no-match is silent. `WebBundleTests` (C#) may need its bundle-structure expectations widened for the new overlay (test-engineer). `docs/ui-checklist.md` §A gains rows for the control cluster, find, and the labels toggle.
- **First interactive bundle overlay:** `pointer-events:auto` on `#controls` means clicks there must not fall through to the canvas — scoped to the cluster's own box, the rest of the page stays `pointer-events:none`/canvas, so graph panning/tapping is unaffected.
