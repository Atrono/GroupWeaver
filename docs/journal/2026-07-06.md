## Decided

- **Smoke promotion bar: solid same-session verification is enough, since smoke is
  advisory/on-demand only** (the `e2e-run` skill before a push), not a CI gate and
  not the nightly `-Tag full` autonomy-loop safety net (WP8/#247, still open) - a
  false positive here costs developer friction, not a masked regression. Revisit
  once WP8's nightly sweep has run these 5 for a few days; demote back to
  `full`-only if any show intermittent `INFRA-DRIVE`/`TIMEOUT` signatures.
- **A failed jury agent doesn't invalidate its surface's findings, but must be
  disclosed inline, not silently dropped or silently presented as fully verified** -
  the fit-audit doc keeps the `audit-1..4` findings (single-juror-verified) rather
  than discarding them, with the gap called out at the doc's top and again at the
  surface header.
