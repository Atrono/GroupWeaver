# 2026-06-28 — Session 36 (wrap-session skill + Stop-hook reminder)

**Phase:** Phase 4 (feedback-driven). v0.4.2 shipped; no in-flight WP at start. The user asked which
Claude skill/agent/workflow would add the most value. Mapped the existing `.claude/` surface (7 agents,
6 skills, 6 rules, 3 hooks; **no commands, no saved workflows**) and the recurring manual rituals via
three parallel Explore passes. The standout gap: the one ritual CLAUDE.md **mandates every session** —
the journal entry → commit → push — was never automated, and "the journal only recovers what reached the
remote." Chose to codify it (option A of four candidates surfaced; B–D parked below).

## Done

- **`wrap-session` skill** (`.claude/skills/wrap-session/SKILL.md`). The session-close counterpart to
  `session-start.ps1`: derive the date + next session number (the same listing the hook uses), draft the
  Done/Decided/Next entry in the house style, stage ONLY the journal, then commit and push as **two
  separate calls**. Encodes the two `guard.ps1` gotchas that silently break the ritual — a combined
  commit+push is mis-flagged as a force-push (the hook's `-match` is case-insensitive, so `\s-f\b` also
  hits `-F`), and AD-write verbs in a commit subject are blocked — plus the same-day refresh rule and the
  "stage explicitly, never `-A`" guard. This entry is the skill's own first dogfood run.
- **`Stop`-hook reminder** (`.claude/hooks/wrap-session-reminder.ps1`, registered in
  `.claude/settings.json`). When `docs/journal/<today>.md` is missing, the Stop hook emits a
  non-blocking `additionalContext` nudge to run `wrap-session` (loop-guarded via `stop_hook_active`;
  reminder only — it never authors). Tested all three branches: silent when today's entry exists,
  valid reminder JSON when absent, silent under the loop guard. Documented in CLAUDE.md's Hooks section.

## Decided

- **A skill, not a Stop hook.** Authoring Done/Decided/Next needs judgment about what the session actually
  did — a hook can only remind, it cannot write the entry. A skill the orchestrator runs at close is the
  right shape; a reminder hook is a possible future complement, never a replacement.
