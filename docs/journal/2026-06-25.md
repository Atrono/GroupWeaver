## Decided

- **Persona value beats mock-parity.** The three deferred v0.4 items were re-ranked by what the
  sysadmin/analyst actually needs to SEE; the audit filter + export won, the editor status bar +
  JSONC highlighting stayed deferred (low persona value / high risk).
- **Whole-page ScrollViewer for the audit view** (WP1) — adding any row tipped the 720px budget over;
  scrolling the page (Auto rows + a `MaxHeight` findings list) is the robust fix vs. pixel-tweaking.

- **Released v0.4.1** (https://github.com/Atrono/GroupWeaver/releases/tag/v0.4.1, PR #173, tag on
  `62d59f8`). SemVer patch — the persona-value audit-screen polish (WP1–WP3). Full cut-release arc:
  4 bump files → local gate green (App 771 / Core 761 incl. live-AD + the 0.4.1 pin smoke) →
  `pack-release 0.4.1` green → **`/security-review` over `v0.4.0..HEAD` CLEAN** (no AD-write; export
  reuses the vetted `ViolationReportExporter` guards + writes only the user-picked path; no exec of
  directory data) → reviewer APPROVE → squash PR #173 → CI green → tag the bump SHA → release.yml
  (provenance attested, repo public) → **public zip verified**: SHA256 matches sidecar
  (`56979ad6…`), `gh attestation verify` exit 0, extracted exe `--check --demo` exit 0
  (`GroupWeaver 0.4.1+62d59f8`, 40 groups). No README-media re-record (audit/settings chrome, not
  the demo-graph GIF).

- **README refresh — Download CTA + scannable overview (#174).** README was ~95% factually
  current for v0.4.1; the problem was shape, not facts. Replaced the wall-of-text "Status"
  paragraph with a download-first layout: a new **Download** section (a `Latest release: vX.Y.Z`

  Releases pointer — the durable home for release history, killing the per-release prose churn and

  Plan and compare / Audit) with every prior feature hoisted into bullets. Preserved all media,
  both guarantees, and the bump touchpoints; re-pointed `cut-release` SKILL.md item 3 from the
  removed "Status line" to the new "Latest release" line. reviewer APPROVE (its two flagged
  omissions — findings-sidebar jump-to-node + the structured matrix rule editor — folded back in),
  CI green. Docs-only.

- **`dotnet format` CLI-resolution flake — fixed at the source (#175).** Root cause: `dotnet
  format` runs as its own child process and re-resolves the dotnet CLI via `DOTNET_HOST_PATH` /
  `DOTNET_ROOT`, NOT the PATH muxer that launched it; after `setup-dotnet` those vars are
  intermittently unset on the windows-2022 runner → "Unable to locate dotnet CLI", cured only by a
  rerun. The prior CI-only `Pin DOTNET_ROOT` step still flaked and never protected the dev box.
  Fix in the **shared gate** `tools/build.ps1`: resolve the absolute dotnet host once
  (`(Get-Command dotnet).Source`, `-ErrorAction Stop`), pin both vars, invoke format via the
  absolute `$dotnet` path. Removed the now-redundant CI step; repointed the cut-release note. Full
  local gate green (build 0 err, format OK, 765 + 717 tests); CI green with the format step clean
  (no "Unable to locate" message). reviewer APPROVE.

## Decided (infra)

- **README is capability-first; release history lives in CHANGELOG/Releases, not README prose.**
  The README describes what the app DOES; version genealogy is linked, not enumerated — durable
  across releases (no prose to bump). Internal phase numbers stay out of the README (PLANNING.md
  territory, no reader value).
- **Fix shared-tool flakes in `tools/build.ps1` (the one gate dev box + CI run), not in a CI-only
  workaround.** A CI-only mitigation can't protect the local gate and tends to drift; the absolute-
  host + env-pin pattern for `dotnet format` is the durable, single-source cure.
