# 2026-06-25 — Session 33 (persona-driven audit-screen polish → WP1–WP3, toward v0.4.1)

**Phase:** Phase 4 (feedback-driven). v0.4.0 redesign (#133) shipped + media refreshed; no in-flight
WP. User asked "what should I focus on", then reframed: *put yourself in an IT sysadmin's /
business-analyst's shoes — what matters, and are the focused functions well-visible?* That reframe
(not mock-parity-for-its-own-sake) drove the whole session.

## Done

- **WP0 — visibility audit (ui-verifier, both themes, vs personas).** The operator loop is mature +
  well-surfaced; the **analyst/auditor** had real gaps: (1) the findings table had sort + triage but
  **no way to slice it** — worst because the Status pills (Open/Ack/Suppressed) shipped un-filterable;
  (2) **no export on the Audit screen** (stranded on the violations sidebar); (3) the **Audit entry
  button** was the least-prominent rail control. Evidence re-scoped v0.4.1 by persona value and
  **dropped** the rule-editor status bar (no persona need) + JSONC highlighting (niche + risky).
- **WP1 — audit findings filter chip strip (#170).** Severity/Status/Rule-class chips, multi-select,
  fail-open; `_allRows` master + filtered `Findings` view; counts over the master; hidden rows
  deselected; rule-class filter prunes across re-threads; "Showing N of M" + distinct no-match state.
  No engine/triage change. **Layout took 3 verify-and-correct iterations** (narrow-column clip →
  vertical overlap → fixed by wrapping the whole audit view in a ScrollViewer + `MaxHeight` list).
  +11 unit + 1 screenshot fixture. ui-verifier PASS, reviewer APPROVE, CI green.
- **WP2 — audit-screen export CSV/HTML (#171).** Reuses the existing `ViolationReportExporter` +
  `IExportFileDialogs` seam end-to-end (no new exporter/dep); exports the **live** report; shell
  threads the connection summary; CTS-cancel-on-Dispose + post-await `IsDisposed` re-guard make a
  dialog-resolved-after-teardown a graceful no-op (a test caught the `ObjectDisposedException`).
  +14 tests. No AD write. ui-verifier PASS, reviewer APPROVE, CI green.
- **WP3 — promote the workspace Audit button (#172).** New `Button.accent-outline` tier (opaque
  accent border+ink, soft hover wash) + a font-independent vector shield-check glyph — a genuine
  third tier, NOT a second filled-accent (no clash with the filled Refresh CTA). New `AccentTextBrush`
  token (deterministic ≥3:1 border, 5.48–6.93:1 both themes). Name/command/tooltip/last-position
  verbatim. +1 regression pin. ui-verifier PASS, reviewer APPROVE, CI green.

## Decided

- **Persona value beats mock-parity.** The three deferred v0.4 items were re-ranked by what the
  sysadmin/analyst actually needs to SEE; the audit filter + export won, the editor status bar +
  JSONC highlighting stayed deferred (low persona value / high risk).
- **Whole-page ScrollViewer for the audit view** (WP1) — adding any row tipped the 720px budget over;
  scrolling the page (Auto rows + a `MaxHeight` findings list) is the robust fix vs. pixel-tweaking.
