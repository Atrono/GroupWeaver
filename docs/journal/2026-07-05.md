## Decided

- **Two-tier E2E driving policy**: Tier A scenarios act only via UIA/WM (prove hittability);
  Tier B may assert via read-only probes but never gets mutation commands. Keeps automation
  honest.
- **Heartbeat over `ProcessFailed`** for WebView2-crash detection - the wrapper doesn't
  expose it; a 10s capability probe with a 3-consecutive-miss threshold is the substitute.
- **Redaction-correct-now**: every WP1/WP2 field touching a DN/path/run-slug routes through
  a typed `Redactor.*` stub today so WP10's real implementation can't silently miss one.
- **Never trust a single channel poll as a settle barrier.** `settings-rethread`'s reviewer
  REJECT showed `Wait-E2eSettled` can report "settled" for the wrong reason (a `state` probe
  colliding with `ProbeStateAsync`'s single-flight guard mid-update reads as "no renderer =
  settled"). Fix pattern going forward: gate the load-bearing assertion on a bounded,
  re-polling check of the actual observable (pixels, file, or repeated state reads), never a
  single sample point - `Wait-CanvasBlobCount` is the reusable shape.
