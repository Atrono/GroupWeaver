## Decided

- **The three remaining fit-audit cheap-wins are correctness/parity, not polish.** A
  dead severity chip, an ambiguous root picker, and rule rationale reachable only from the
  Audit screen are trust/usability gaps; closing them brings the explore path to parity
  with the Audit screen. None warranted an ADR (reuse of existing patterns + copy on
  existing surfaces; issue + commit suffice).
- **Took the reviewer's DN backslash-run-parity nice-to-have inline** (not deferred) -
  cheap and correct for a reusable converter, so its doc contract stays literally true.
- **Media-currency is a NON-BLOCKING git-timestamp reminder, not a pixel gate.** Rendering
  is non-deterministic across GPU/software modes and a source change doesn't always alter the
  media, so an exact-hash CI gate would be flaky/false-positive. The check warns (exit 0;
  `-Strict` opt-in for future CI) so a human judges. Known limitation: it's `docs/media`-commit
  granularity, so refreshing any one asset marks all current - acceptable for a reminder; a
  per-asset-group refinement is the future option if it ever masks a real gap.
- **The reusable audit topology is a Workflow, not a skill - and audits stay read-only.** A
  prose skill would mostly repackage always-loaded CLAUDE.md (modest gain); encoding the
  agent fan-out as a Workflow removes the per-audit re-derivation (net-new). The workflow
  writes nothing and files no issues - the orchestrator persists results - so the
  read-only-product invariant holds even in the tooling. `args` cross the runtime boundary
  JSON-stringified, and directly-awaited terminal agents sit outside `pipeline`/`parallel`
  isolation, so they must be defended explicitly (parse + try/catch) - both caught by
  testing the workflow before trusting it.
- **Verify a fit-audit finding before implementing it.** Lever 4's lever text asked for
  "per-hue ink"; the implementer computed the actual WCAG ratios, found white already clears
  4.5:1 on every kind fill, and correctly skipped the ink map - the real defect was fill-vs-page
  (1.4.11), fixed by the ring alone. Audit findings are hypotheses, not specs.
- **A `main` CI run (journal commit `f1f085a`) red-flaked on `dotnet format` "Unable to locate
  dotnet CLI"** - a transient runner PATH hiccup (the build itself passed; the commit had no
  `.cs`). First occurrence; self-cured on the next push. Harden `build.ps1`'s dotnet discovery
  only if it recurs.
