# 2026-06-27 — Session 35 (scope-summary card reframe → ruleset surfaced)

**Phase:** Phase 4 (feedback-driven). v0.4.2 shipped; no in-flight WP at start. The user flagged that
the no-selection right-rail "Scope summary" card and the graph legend show "the same" kinds, and asked
whether the card earns its place. Audited it: of the card's four elements, the **per-kind tally**
duplicated the always-on graph legend's Kinds section and the **severity tally** duplicated the findings
chip strip directly above it in the same rail — only the object/edge totals + the "click a node" hint
were unique. Meanwhile the one orientation fact the workspace surfaced **nowhere** was the **active
ruleset name** (root DN is already in the status bar, demo/live in the top strip). So: reframe, not delete.

## Done

- **#186 / PR #187 (squash `a539ee4`).** Dropped the per-kind tally + severity tally from
  `ScopeSummaryCard`; added a `Ruleset: <name>` line — new computed `WorkspaceViewModel.RulesetName`,
  `OnPropertyChanged(nameof(RulesetName))` raised in `ApplyRulesetAsync` **before** its early-return guard
  so a renderer-less settings-apply still notifies. Retired `ScopeKindTally` + deleted dead `KindTallyRow`;
  dropped the now-unused `rules:` xmlns from `WorkspaceView.axaml`. `GraphSummary` (and its test-pinned
  format) untouched; `UpdateGraphSummary` reduced to the totals. New unit test pins `RulesetName` initial
  value + live update/notification. Rewrote the `ui-checklist.md` scope-summary row; appended the dated
  ADR-022 D5 addendum. Gate green (801 App + 761 Core), reviewer APPROVE, ui-verifier PASS (incl.
  2560×1080 ultrawide — no void), CI green.

## Decided

- **Redundant filler is worse than honest whitespace.** The no-selection rail should carry the one fact
  shown nowhere else — the ruleset judging the findings — not echo the legend/sidebar. Recorded as the
  2026-06-27 ADR-022 D5 addendum (refines, does not supersede, the accepted decision).
