# 2026-06-23 — Session 26 (user-reported Back-nav crash fixed end-to-end: #120 / PR #121 / ADR-024)

**Phase:** Phase 4 (feedback-driven). Single work package, full arc: reproduce → fix →
test → ADR → reviewer APPROVE → squash PR → CI green → issue closed. `main` at `527ec92`,
all green on CI (App.Tests 596/596, Core 700/700).

## Done

- **#120 Back-button crash (PR #121, ADR-024).** User reported the app crashes pressing
  Back on "some functions." Root cause: the shell swaps steps via one `ContentControl`;
  Workspace/Plan/Gap each own a `CytoscapeGraphRenderer` with one `NativeWebView`, and the
  leaving view never released it → on Back the new view re-parented a control still owned by
  the old `GraphHost` → `InvalidOperationException: already has a visual parent` on the
  measure pass. RootPicker→Connect has no graph, hence "some functions."
- **Fix (3 layers):** views release `GraphHost.Content` on `DetachedFromVisualTree` +
  stale-parent guard on mount (the crash fix); renderer re-navigates + replays the cached
  last render on re-attach (NativeControlHost recreates the WebView2 child blank); all
  render/focus/export command paths made never-throw (InvokeScript fault → RendererError).
- **Reproduction-first** via a NEW windowed `--demo` smoke `tools/smoke-back-nav.ps1`
  (UIA + posted-WM driving, re-resolves the Chromium child HWND per swap). Captured the exact
  exception, then drove the fix to PASS on both Design plan→Back and the Gap round-trip.
- **Regression now in CI:** the crash is a pure Avalonia parent conflict (not WebView2-only),
  so `BackNavigationStepSwapTests` reproduces it headlessly with a real-`Control` fake (RED→GREEN),
  plus `RendererFaultNeverCrashesTests`. Opt-in `FakeGraphRenderer.WithRealSurface()` (default
  unchanged). 9 new tests green.

## Decided

- **Re-render-on-reattach over BeginReparenting.** Restoring the graph by re-navigate+replay
  (self-contained, robust) rather than the package's across-turn `BeginReparenting` (preserve
  the live page) — viewport-preservation is a noted future enhancement (ADR-024 Alternatives).
- **Gate the replay by PROBING `InvokeScript`**, not by events. Resetting `_ready` and a
  `NavigationCompleted` gate both lost to stale, dispatcher-queued bridge messages on the
  *second* re-attach ("Unable to invoke script before any page was loaded"). Polling the real
  precondition (page accepts script + `window.bridge.dispatch` exists) is race-immune.

## Next

- Optional follow-up issue: viewport-preserving Back via `BeginReparenting` (no reload flash),
  and renderer `IDisposable` to retire the pre-existing never-disposed-renderer leak.
- Local gate caveat persists: 12 headless view-realization tests fail on this box (GPU driver
  absent after rebuild) but are green on CI — rely on CI, or reinstall the Intel driver.

## Done

- **#122 viewport-preserving Back (PR #125, ADR-025).** Built the 7-slice parking-lot design:
  renderer `IDisposable` → `GraphSurfaceCoordinator` + hidden always-attached `ParkingLot` panel
  → wired via the `IExportFileDialogs` MainWindow→VM seam → views `Mount` (re-render kept as the
  fallback) + child-scoped detach guard → `ShellViewModel` parks the Back-target surface BEFORE
  `CurrentStep` reassign + reclaims abandoned Gap/superseded-Plan surfaces (bounded ≤ Workspace +
  current Plan). A leaving graph surface is now PARKED (kept rooted, hidden) and moved back via
  one synchronous `BeginReparenting(true)` hop, so its WebView2 page + cytoscape viewport survive
  Back — no reload flash.
- **Verified every tier:** 10 new headless tests (parking choreography, bounded count 0→1→2→1→0,
  abandoned renderers disposed) + coordinator unit tests, all green; **windowed smoke PASS** —
  same `Chrome_RenderWidgetHostHWND` across BOTH Back paths (the real-WebView2 proof, extended
  `tools/smoke-back-nav.ps1` with a largest-visible-child resolver since a parked hidden child now
  coexists); reviewer APPROVE; CI green (App 606/606, Core 700/700).
- **Released v0.3.1** (https://github.com/Atrono/GroupWeaver/releases/tag/v0.3.1) — patch (UX
  refinement, no new features). Full cut-release arc again: bump → gate → pack `0.3.1` → security
  review over `v0.3.0..HEAD` CLEAN → reviewer → PR #126 → CI → tag `71d6e00` → release.yml
  (attested) → public zip verified (SHA256 `527e8255…`, attestation exit 0, exe `GroupWeaver
  0.3.1+71d6e00`).

## Decided

- **Patch (0.3.1), not minor.** #122 refines existing Back behavior + retires a leak; no new
  user-facing features → patch (matches the v0.2.1/v0.3.x cadence).
- **Renderer teardown = detach, not `NativeWebView.Dispose`** (it isn't `IDisposable`): detaching
  from the host is exactly how `NativeControlHost` frees the WebView2 child (ADR-024). `_disposeCts`
  only unwinds the re-navigate probe loop; in-flight commands unwind via the `_disposed` guard.
- **ASCII-only smoke script.** PS 5.1 reads no-BOM files via the ANSI codepage; the Edit tool
  strips BOMs, so em-dash/arrow glyphs mojibaked and broke parsing — converted to ASCII (durable).
