## Decided (continued)

- **A filter escaper is now a real thing this codebase needs, not a
  hypothetical** - ADR-039 is the first ADR admitting an LDAP filter composed
  from directory data; `LdapFilter.Escape` is the one place that happens and
  every future filter-composing change must route through it.
- **Kerberos sealing+signing over LDAPS** - works wherever Kerberos already
  works (no per-DC cert dependency), so it's now the unconditional floor, not a
  configurable option (ADR-040 D3).
- **God-class decomposition ships as reviewed slices, not one big rewrite** -
  each cluster gets its own PR/reviewer pass; stopping after one slice per
  session is fine, the backlog issue tracks what's left.
- **Not every identified cluster should be extracted** - a collaborator that
  would just be VM-owned state + ceremony (run-history, blocked on
  CommunityToolkit's same-class `nameof`/AXAML-binding requirements) is better
  left in place and documented than extracted for extraction's sake.
