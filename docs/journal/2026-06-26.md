# 2026-06-26 — Session 34 (senior UI/UX review polish → v0.4.2)

**Phase:** Phase 4 (feedback-driven). v0.4.1 shipped; no in-flight WP. User asked, from a
senior UI/UX-developer lens, *"is the frontend good, should I change something?"* I audited all
six surfaces first-hand (graph overview, gap, Audit, Workspace+rail, Connection, Settings) +
the full design system. Verdict: genuinely strong (real token system, WCAG 2.2 AA throughout,
colourblind-redundant dual-encoding, honest tri-state) — not a templated default. So this was a
polish pass, not a rescue: four refinements, each its own branch → PR → reviewer → CI → squash.

## Done

- **WP-A — graph overview edge-fade (#176, PR #177, ADR-029).** The default fit view was a
  full-opacity edge hairball (and the README hero). Now plain non-diff edges fade to opacity
  0.15 at/near the fit zoom and return on zoom-in — binary class toggle, hysteresis-gated (the
  `cy.batch` fires only on a threshold crossing; `fitZoom`-relative so it auto-scales), instant
  (no transition ⇒ #88 motion counters stay 0), gap/diff view excluded wholesale (`isDiffGraph`).
  Pinned in verify.mjs (+20 asserts). **Re-pointed the README hero** `graph-explore.png` from the
  zoomed `graph-focus` frame to the now-clean faded `graph-overview` constellation (record-demo-media
  skill mapping updated to match).
- **WP-B — workspace rail (#178, PR #179).** The findings sidebar got only 2/5 of the rail
  (~1 finding visible). Now a persisted, resizable findings/detail split (new
  `UiState.RailFindingsFraction`, default 1:1, clamped [0.2,0.8]) via a hairline GridSplitter
  (one-way rows + code-behind `DragCompleted` write-back = jitter-free). Refresh's accent is now
  conditional on `IsNodeSelected` so the empty state's loudest control isn't a dead button.
- **WP-C — audit de-dup (#180, PR #181).** The "Rule classes" categories pane and the strip's
  RULE CLASS chip group rendered the same list twice. Realized the team's own *"click-to-filter is
  WP5d/e"* TODO: the categories rows ARE the rule-class filter now (`ToggleCategory`, `AuditCategoryRow`
  → ObservableObject with `IsActive`), `RuleClassChips` removed, single source = `_ruleClassFilter`.
  Plus a scrollbar-gutter margin so the 4th count tile no longer clips at 1280.
- **WP-D — minor polish (#182, PR #183).** Progressive connect-ring fade (520 ring most present
  ~9%, fading outward — still neutral-white separator language); full-row Settings rules hover wash
  tying name↔severity. ui-verifier caught a real Avalonia value-precedence bug (inline
  `Background="Transparent"` LocalValue swallowed the `:pointerover` setter) — fixed by moving the
  rest fill into a base Style.
- **Released v0.4.2** (PR #184, tag on the bump commit `7a6dfa3`). Four bump files → local gate
  green (800 App / 761 Core incl. live-AD + the 0.4.2 pin smoke) → `pack-release 0.4.2` green →
  **`/security-review` over `v0.4.1..HEAD` CLEAN** (no new untrusted-input→sink; no AD-write; all
  five threat surfaces clear) → release.yml green → **public artifact verified**: SHA256 matches
  sidecar, `gh attestation verify` exit 0, extracted exe `--check --demo` prints `GroupWeaver
  0.4.2+7a6dfa3 … connected, 40 groups loaded`.

## Decided

- **Persona/quality reframe over mock-parity:** the audit started from "what would a senior UI/UX
  dev change," not a mockup. Two of the four refinements turned out to be the team's own deferred
  TODOs (the audit categories click-to-filter; the action-hierarchy guard). Left settled decisions
  alone (ADR-027 shapes-not-corner-badges, deferred JSONC highlighting).
- **The README hero is now the OVERVIEW frame, not the focus frame** — the edge-fade is what makes
  the overview a sellable hero; the zoomed `graph-focus` stays the `[S:graph-focus]` checklist frame.
