## Done

- **#232 / PR #233 (squash `d6db4f5`).** ROOT-CAUSED the recurring `dotnet format`
  "Unable to locate dotnet CLI" (exit 4) CI flake: it is an UPSTREAM race in
  dotnet-format's own CLI probe (spawns the dotnet host, reads stdout via async
  redirection; `WaitForExit` vs `BeginOutputReadLine` loses the output on loaded
  runners - dotnet/format#2000, persisting on 8.x per dotnet/sdk#44957). That is why
  #110's env pinning provably could not cure it. Fix: `build.ps1` retries the format
  step ONCE iff exit 4 AND the literal probe message (Tee-Object capture); a real
  format diff exits 2 without the message and the retry re-runs the full verification,
  so it can never false-pass. Stub-proven both arms; comments in build.ps1/ci.yml
  de-overclaimed; `#Requires` bumped 7 -> 7.2 (reviewer note: pre-7.2 stderr semantics).
- **#230 / PR #234 (squash `f817657`).** Keyboard-focus continuity across the
  focus-mode chrome melt - the app's FIRST programmatic-focus pattern (ADR-022 third
  addendum): MainWindow observes `IsFocusMode` via the existing OnOpened
  PropertyChanged seam; entry parks focus unconditionally on the surviving seam chevron
  via `Focus(NavigationMethod.Tab)` (so `:focus-visible` fires and the ADR-033 accent
  ring RENDERS - the ring doubles as the way-back affordance; never the WebView2 HWND,

  ADR-022 Consequences). TDD: headless proved the defect is focus->NULL; 3 new + 1
  extended AvaloniaFacts drive real Enter/Esc/F key dispatch. Gate green (1029 App +
  858 Core), reviewer APPROVE, ui-verifier PASS (pixel-sampled #A99BFF 2px ring on the
  chevron, 6.93:1, ring absent in non-focus control frames), CI green.

## Decided

- **The #232 flake is upstream and unfixable locally except by retrying** - established
  during planning (web evidence, not guesswork), so the slice shipped a signature-gated
  retry instead of another env-pinning round; recorded in build.ps1's comment so the
  next recurrence isn't re-root-caused. The retry did not fire in any of today's three
  CI runs (expected: intermittent).
- **Programmatic focus is view-owned and parks on the designated surviving affordance**
  (ADR-022 third addendum) - the pattern future focus-moving features copy;
  `NavigationMethod.Tab` is load-bearing (it is what makes the ADR-033 ring render).
- **Near-metamer token changes need a value-pinning test, not visual QA** - #227's
  dark/light AccentSoft values are indistinguishable composited over light backgrounds
  (<1.5/channel), so the live re-tone test is the only regression guard; ui-verifier
  proved the rest by per-pixel A/B against a pre-change worktree baseline.
